Bootstrap拥有大量方便实用的组件或插件

组件

  • 无数可复用的组件，包括字体图标、下拉菜单、导航、警告框、弹出框等更多功能。

字体图标

一波来自 Glyphicon Halflings 的字体图标

图标类不能和其它组件直接联合使用且只能应用在不包含任何文本内容或子元素的元素上。

应该创建一个嵌套的 <span> 标签，并将图标类应用到这个 <span> 标签上。

通过aria-hidden或aria-label属性确保图标含义能被辅助设备识别。

下拉菜单

用于显示链接列表的可切换、有上下文的菜单。下拉菜单的 JavaScript 插件让它具有了交互性。

将下拉菜单触发器和下拉菜单都包裹在 .dropdown (.dropup)里，或者另一个声明了 position: relative; 的元素。然后加入组成菜单的 HTML 代码。

.dropdown-menu-right 在 .dropdown-menu 中添加可以让菜单右对齐。

.dropdown-header 在下拉菜单中添加标题来标明一组动作。

.divider 为下拉菜单添加一条分割线，用于将多个链接分组。

.disabled 禁用相应的菜单项。

实例见http://v3.bootcss.com/components/#dropdowns-example

按钮组

通过按钮组容器把一组按钮放在同一行里。通过与按钮插件联合使用，可以设置为单选框或多选框的样式和行为。

当为 .btn-group 中的元素应用工具提示或弹出框时，必须指定 container: ‘body’ 选项，这样可以避免不必要的副作用（例如工具提示或弹出框触发时，会让页面元素变宽和/或失去圆角）。

按钮组和工具栏应给定一个明确的label标签

具体更多见http://v3.bootcss.com/components/#btn-groups-single

Bootstrap规范

属性顺序

HTML 属性应当按照以下给出的顺序依次排列，确保代码的易读性。

  1. class
  2. id,name
  3. data-*
  4. src,for,type,href,value
  5. title,alt
  6. role,aria-*

声明顺序

相关的属性声明应当归为一组，并按照下面的顺序排列：

  1. Positioning(position, top, z-index …)
  2. Box model(display, float, width …)
  3. Typographic(font, line-height, color, text-align …)
  4. Visual(backgound-color, border …)
  5. Misc(opacity)

杂项

  • 减少标签的数量
  • 避免使用JavaScript生成的标签
  • 不要使用 @import
  • 将媒体查询放在尽可能相关规则的附近
  • 对于只包含一条声明的样式，建议将语句放在同一行。对于带有多条声明的样式，还是应当将声明分为多行。
  • 在需要显示地设置所有值的情况下，应当尽量限制使用简写形式的属性声明。过度使用简写形式的属性声明会导致代码混乱，并且会对属性值带来不必要的覆盖从而引起意外的副作用。
  • class 名称中只能出现小写字符和破折号（dashe）
  • class 名称应当尽可能短，并且意义明确。避免过度任意的简写
  • 基于最近的父 class 或基本（base） class 作为新 class 的前缀。
  • 使用 .js-* class 来标识行为（与样式相对），并且不要将这些 class 包含到 CSS 文件中。
  • 对于通用元素使用 class ，这样利于渲染性能的优化
  • 对于经常出现的组件，避免使用属性选择器
  • 选择器要尽可能短，并且尽量限制组成选择器的元素个数，建议不要超过 3 。
  • 以组件为单位组织代码段
